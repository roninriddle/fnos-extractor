# 修复：无法搜索挂载目录下的压缩包

## 问题描述
用户无法搜索到挂载目录下的压缩包，导致扫描功能失效。

## 根本原因
1. **权限问题**：在挂载目录中，某些子目录可能存在权限限制，导致 `rglob` 方法抛出异常
2. **异常处理不足**：原代码在遇到权限错误时会中断扫描，无法继续处理其他文件
3. **缺少备选方案**：没有在主方法失败时使用备选的遍历策略

## 实施的修复

### 1. 改进 `find_all_archives()` 函数

**改进点**：
- 添加目录存在性和类型检查
- 为 `rglob` 操作添加内层异常处理，跳过无法访问的文件而不中断扫描
- 当 `rglob` 失败时，使用 `os.walk()` 作为备选方案
- 添加详细的日志记录便于调试

```python
# 关键改进：
1. 双层异常处理（外层和内层）
2. rglob 失败时自动降级到 os.walk
3. 跳过权限错误的文件而继续扫描
4. 详细的日志记录所有错误和跳过的文件
```

### 2. 改进 `scan_subdirectories()` 函数

**改进点**：
- 添加目录验证
- 在 `rglob` 失败时使用 `os.walk` 作为备选
- 继续处理下一个子目录而不中断

### 3. 改进 `/api/scan` 路由

**改进点**：
- 添加目录权限检查（`os.access()`）
- 包装压缩包扫描和子目录扫描在 try-catch 中
- 处理单个压缩包分析时的异常，防止单个文件错误导致整体失败
- 添加详细的错误日志

### 4. 错误处理策略

```
主尝试 (rglob) 
    ↓ 失败
备选方案 (os.walk) 
    ↓ 失败
记录错误并继续处理其他文件
```

## 技术细节

### os.walk 备选方案优势
- **权限友好**：不会因单个文件权限不足而停止
- **跨平台兼容**：在所有操作系统上表现一致
- **可控性**：可以过滤无法访问的目录

```python
# 关键代码示例
dirnames[:] = [d for d in dirnames if os.path.exists(os.path.join(dirpath, d))]
```

## 测试场景
该修复应该解决以下场景：
1. ✅ 挂载目录中某些子目录权限受限
2. ✅ 符号链接导致的访问问题
3. ✅ 混合权限的目录结构
4. ✅ NAS 或远程挂载的目录
